\section{Introduction}
\label{sec:intro}

\begin{itemize}[leftmargin=1.4em,itemsep=5pt,topsep=2pt]

  \item \textbf{Fixed-size memory, edited online.} Linear attention replaces the
  growing key--value cache of softmax attention with a fixed-size recurrent state
  $S_t \in \R^{d_k \times d_v}$ that is \emph{written} as tokens stream in and
  \emph{read} by the query \citep{katharopoulos2020transformers},
  \begin{equation}
    \text{(write)}\quad S_t = S_{t-1} + k_t v_t^{\tr},
    \qquad\qquad
    \text{(read)}\quad o_t = S_t^{\tr} q_t .
    \label{eq:linattn}
  \end{equation}
  The design question is how to \emph{edit} this store online --- what to write and
  what to forget. This plain additive write only ever \emph{adds} associations:
  nothing is removed and non-orthogonal keys interfere, so the store saturates and
  reads degrade.

  \item \textbf{DeltaNet: error-correcting writes.} The delta rule
  \citep{schlag2021linear,yang2024parallelizing} reads what is stored at $k_t$ and
  writes back only the correction,
  \begin{equation}
  \begin{aligned}
    S_t
      &= S_{t-1} + \beta_t k_t\,(v_t-S_{t-1}^{\tr}k_t)^{\tr}
        \qquad\text{(canonical residual form)}\\
      &= (I-\beta_t k_t k_t^{\tr})\,S_{t-1} + \beta_t k_t v_t^{\tr},
  \end{aligned}
    \label{eq:deltanet}
  \end{equation}
  so a repeated key is \emph{overwritten} rather than accumulated. It still never
  forgets what \emph{other} keys hold.

  \item \textbf{Gated DeltaNet: forgetting by decay.} DeltaNet corrects the current key
  but removes nothing else, so stale associations linger and the state drifts over long
  contexts. To actually \emph{forget}, Gated DeltaNet shrinks the whole state by a
  data-dependent decay $\alpha_t\in(0,1)$ before each write \citep{yang2024gdn},
  \begin{equation}
  \begin{aligned}
    S_t
      &= \alpha_t S_{t-1}
         + \beta_t k_t\,(v_t-(\alpha_t S_{t-1})^{\tr}k_t)^{\tr}
        \qquad\text{(canonical residual form)}\\
      &= \alpha_t\,(I-\beta_t k_t k_t^{\tr})\,S_{t-1} + \beta_t k_t v_t^{\tr},
  \end{aligned}
    \label{eq:gdn}
  \end{equation}
  so untouched memories fade geometrically: the state stays bounded and recent tokens
  weigh more. But a single scalar $\alpha_t$ per head fades \emph{every} channel at the
  same rate --- it cannot hold some memories long while dropping others fast.

  \item \textbf{KDA: multi-timescale forgetting.} But a useful memory needs several
  horizons at once --- a name or topic kept for thousands of tokens, a local cue dropped
  immediately. KDA replaces the scalar with a diagonal $D_t=\diag(\alpha_t)$, giving each
  channel its own decay rate \citep{kimiteam2025kimilinear},
  \begin{equation}
  \begin{aligned}
    S_t
      &= D_t S_{t-1}
         + \beta_t k_t\,(v_t-(D_t S_{t-1})^{\tr}k_t)^{\tr}
        \qquad\text{(canonical residual form)}\\
      &= (I-\beta_t k_t k_t^{\tr})\,D_t\,S_{t-1} + \beta_t k_t v_t^{\tr},
  \end{aligned}
    \label{eq:kda}
  \end{equation}
  so one head spans a \emph{mixture of timescales} --- the most expressive forgetting in
  the family, and the baseline we build on.

\end{itemize}

\section{Associative Memory as a State-Space Model}
\label{sec:memory-kalman}

\subsection{Formulation}
\label{sec:memory-formulation}

The recurrent state in linear attention is not just a cache compression; it is an
\emph{estimate} of an associative memory. At time $t$, the memory is a matrix
$S_t\in\R^{d_k\times d_v}$ that defines the associative map
\begin{equation}
  \mc M_t\colon \R^{d_k}\to\R^{d_v},\qquad \mc M_t(k)=S_t^{\tr}k .
  \label{eq:assoc-map}
\end{equation}
The query read is $o_t=\mc M_t(q_t)=S_t^{\tr}q_t$. The streaming token provides one
supervised measurement of this map: under key $k_t$, the memory should return value $v_t$,
i.e.\ $\mc M_t(k_t)\approx v_t$. This is exactly the shape of a state-space model: an
unobserved state evolves over time, and each token reveals a noisy linear projection of it.

\paragraph{Latent state.}
Let $S_t^\star$ be the latent associative map the recurrent state is trying to track. The
transition describes how stored associations persist, decay, or drift before the next
measurement arrives:
\begin{equation}
  S_t^\star = D_t S_{t-1}^\star + W_t,\qquad W_t\sim\N(0,Q_t).
  \label{eq:mem-update-dynamics}
\end{equation}
The operator $D_t$ is the process model: it is the formal object corresponding to forgetting
under information drift, topic drift, and other non-stationarity in the stream. For DeltaNet
$D_t=I$; for Gated DeltaNet $D_t=\alpha_t I$; for KDA $D_t=\diag(\alpha_t)$. The process
noise $W_t$ represents memory drift not explained by deterministic decay, while $Q_t$ is
its covariance.

\paragraph{Observation.}
The current key reads one direction of the latent map. The observed value is
\begin{equation}
  v_t = S_t^{\star\tr}k_t + e_t,\qquad e_t\sim\N(0,r I).
  \label{eq:mem-update-observation}
\end{equation}
Thus $k_t^{\tr}$ is the observation map and $v_t$ is the measurement. The noise term $e_t$
captures the part of the token value that should \emph{not} be treated as a clean memory
target. In an LLM, $v_t$ is a contextual feature produced from a noisy token embedding,
local syntax, position, layer mixing, and finite model capacity; not all of it is a stable
fact that can be stored under key $k_t$. The covariance $rI$ sets how reliable this value
observation is: small $r$ means trust the token and write strongly, while large $r$ means
treat much of $v_t$ as context-specific noise.

\paragraph{Filtering problem.}
The state-space model asks for an online filter: given the previous memory estimate and the
new noisy measurement $(k_t,v_t)$, update the associative memory so it tracks the
non-stationary stream. Forgetting is not an auxiliary gate added to attention; it is part of
the state dynamics that predict how memory changes before the next measurement arrives.


% =============================================================================
\subsection{Kalter Filtering}
\label{sec:kalter-filtering}

The state-space model in \S\ref{sec:memory-formulation} gives a principled solution to the
filtering problem: combine the prior memory estimate with each new measurement so that the
current state estimate has minimal uncertainty. Under the linear--Gaussian assumptions in
\eqref{eq:mem-update-dynamics} and \eqref{eq:mem-update-observation}, this becomes the
Kalman filtering update. The filter carries $S_t$ as the current memory estimate, and a
covariance $P_t\in\R^{d_k\times d_k}$ tracks uncertainty over key directions in that
estimate.

Tracking $P_t$ is necessary because each token observes the associative map only at one key
direction $k_t$, and the value $v_t$ is noisy. After many tokens, some key directions are
well supported by past measurements, while others remain uncertain or have become stale
because of drift. High uncertainty means the current token should be allowed to edit the
memory strongly; low uncertainty means the prior memory should resist a noisy measurement.

\paragraph{Predict: forgetting before writing.}
Before observing $(k_t,v_t)$, the filter predicts how the state evolves from step $t-1$ to
step $t$:
\begin{equation}
  \widehat{S}_t = D_t S_{t-1},
  \qquad
  \widehat{P}_t = D_tP_{t-1}D_t^{\tr}+Q_t .
  \label{eq:kalter-predict}
\end{equation}
This is the formal role of forgetting. The transition $D_t$ moves the associative memory
forward in time, while $Q_t$ increases uncertainty for drift that the transition cannot
explain.

\paragraph{Read: compare the prediction to the token.}
The predicted value under the current key is
\begin{equation}
  \hat v_t = \widehat{S}_t^{\tr}k_t,
  \qquad
  \delta_t = v_t-\hat v_t .
  \label{eq:kalter-innovation}
\end{equation}
The vector $\delta_t$ is the innovation: the part of the observed value that the predicted
memory does not explain.

\paragraph{Objective: minimize posterior uncertainty.}
After observing $(k_t,v_t)$, we update the predicted memory state by writing the innovation
back into memory. The only remaining choice is the write direction $\kappa_t$: it controls
which key-space direction receives the residual. For a candidate $\kappa_t$, the memory
update has the form
\begin{equation}
  S_t=\widehat{S}_t+\kappa_t\big(v_t-\widehat{S}_t^{\tr}k_t\big)^{\tr}.
  \label{eq:kalman-residual-write}
\end{equation}
The Kalman objective chooses the write direction that leaves the updated memory estimate as
certain as possible, i.e.\ the one that minimizes posterior uncertainty after the
measurement:
\begin{equation}
  \kappa_t^\star
  =
  \arg\min_{\kappa_t}
  \Trace\!\left(P_t(\kappa_t)\right),
  \qquad
  P_t(\kappa_t)
  =
  (I-\kappa_t k_t^{\tr})\widehat{P}_t(I-\kappa_t k_t^{\tr})^{\tr}
  + r\,\kappa_t\kappa_t^{\tr}.
  \label{eq:kalman-tracking-objective}
\end{equation}

\paragraph{Update: write the uncertainty-weighted residual.}
Solving the objective in \eqref{eq:kalman-tracking-objective} gives the Kalman gain, which
decides how much to trust the token relative to the current memory:
\begin{equation}
  \kappa_t = \frac{\widehat{P}_t k_t}{r+k_t^{\tr}\widehat{P}_t k_t}.
  \label{eq:kalter-gain}
\end{equation}
The optimal filtering update writes only the innovation, along this uncertainty-weighted key
direction:
\begin{equation}
  S_t = \widehat{S}_t+\kappa_t\delta_t^{\tr},
  \qquad
  P_t=(I-\kappa_tk_t^{\tr})\widehat{P}_t .
  \label{eq:kalter-update}
\end{equation}
Finally, the layer reads from the filtered memory,
\begin{equation}
  o_t = S_t^{\tr}q_t .
  \label{eq:kalter-read}
\end{equation}
This filtering update decomposes memory editing into the three operations the model needs:
predict the drifting memory, measure the prediction error at the current key, and write back
the residual in proportion to uncertainty.

\begin{tcolorbox}[
  title={Kalman Optimal Update},
  colback=blue!7,
  colbacktitle=blue!22,
  colframe=blue!45!black,
  coltitle=black,
  fonttitle=\bfseries,
  boxrule=0.4pt,
  arc=1pt,
  left=5pt,
  right=5pt,
  top=5pt,
  bottom=5pt
]
Given the memory estimate $S_{t-1}$ and uncertainty $P_{t-1}$ before token $t$, the
Kalman filter first predicts how the memory drifts under the process model $D_t$ and
process noise $Q_t$. After observing the key--value pair $(k_t,v_t)$, it writes only the
value residual $v_t-\widehat{S}_t^{\tr}k_t$ along the uncertainty-weighted direction
$\kappa_t$.
\begin{equation}
\begin{aligned}
  \text{predict:}\quad
  \widehat{S}_t &= D_t S_{t-1},
  &
  \widehat{P}_t &= D_t P_{t-1}D_t^{\tr}+Q_t,
  \\[3pt]
  \text{update:}\quad
  S_t &= \widehat{S}_t+\kappa_t\big(v_t-\widehat{S}_t^{\tr}k_t\big)^{\tr},
  &
  P_t &= (I-\kappa_t k_t^{\tr})\widehat{P}_t .
  \\[3pt]
  \text{where}\quad
  \kappa_t &= \frac{\widehat{P}_t k_t}{r+k_t^{\tr}\widehat{P}_t k_t}.
\end{aligned}
\label{eq:kalman-optimal-update-box}
\end{equation}
\end{tcolorbox}

\subsection{Delta-rule models as fixed-gain Kalman filters}
\label{sec:delta-family-kalman}

The update above separates two modeling choices: the transition $D_t$, which predicts how
the memory changes before the token is written, and the gain $\kappa_t$, which decides the
key-space direction and strength of the residual write. Delta-rule linear attention is the
fixed-gain special case of this filter. If we discard the covariance recursion and use an
isotropic surrogate $\widehat{P}_t\approx c_t I$ for the predicted uncertainty, then
\begin{equation}
  \kappa_t
  =
  \frac{\widehat{P}_t k_t}{r+k_t^{\tr}\widehat{P}_t k_t}
  \approx
  \frac{c_t}{r+c_t\|k_t\|_2^2}\,k_t
  \equiv
  \beta_t k_t .
  \label{eq:fixed-gain-approx}
\end{equation}
For normalized keys, the exact Kalman gain therefore collapses to the scalar write strength
$\beta_t$ used by the delta-rule family. Substituting $\kappa_t=\beta_t k_t$ into the
Kalman update gives the shared fixed-gain residual form
\begin{equation}
\begin{aligned}
  S_t
  &= D_t S_{t-1}
     + \beta_t k_t\big(v_t-(D_tS_{t-1})^{\tr}k_t\big)^{\tr} \\
  &= (I-\beta_t k_t k_t^{\tr})D_tS_{t-1}+\beta_t k_t v_t^{\tr}.
\end{aligned}
\label{eq:fixed-gain-delta-family}
\end{equation}
DeltaNet, Gated DeltaNet, and KDA are obtained by choosing different process models:
\begin{equation}
  \text{DeltaNet: }D_t=I,\qquad
  \text{GDN: }D_t=\alpha_t I,\qquad
  \text{KDA: }D_t=\diag(\alpha_t).
  \label{eq:delta-family-transitions}
\end{equation}
Thus the three models share the same measurement model and residual write; they differ only
in how the memory is predicted before the write. Relative to the Kalman optimal update, they
freeze the uncertainty dynamics and replace the adaptive gain with the fixed first-order
direction $\beta_t k_t$. This does not require assuming $Q_t=0$; rather, the delta-rule
approximation ignores the covariance prediction
$\widehat{P}_t=D_tP_{t-1}D_t^{\tr}+Q_t$ altogether, so the gain no longer depends on the
process model $D_t$ or the drift uncertainty $Q_t$.

\section{Kalman Linear Attention}
\label{sec:kla}

The Kalman optimal update in \eqref{eq:kalman-optimal-update-box} is the target, but the
exact recursion is not directly compatible with fully parallel linear attention. Parallel
training relies on a scan form
\begin{equation}
  S_t=A_tS_{t-1}+b_t,
  \label{eq:parallel-affine-scan}
\end{equation}
where the per-token coefficients $A_t$ and $b_t$ are input-only, or are computable from
associative prefix statistics. Exact Kalman filtering breaks this structure because
$P_t$ follows a Riccati recursion and the gain $\kappa_t$ depends on the accumulated
posterior uncertainty. Kalman Linear Attention therefore asks which parts of the optimal
update can be relaxed while preserving the scan.

\paragraph{Direction A: richer process models.}
One route is to keep the residual write fixed or cheap, but improve the prediction
$\widehat{S}_t=D_tS_{t-1}$. This treats the main modeling problem as learning how memory
should drift before the next token is written.
\begin{enumerate}[leftmargin=1.8em,itemsep=3pt,topsep=2pt,label=\textbf{A\arabic*.}]
  \item \textbf{Diagonal transitions.} Use $D_t=\diag(\alpha_t)$, as in KDA. This is the
  simplest fully parallel multi-timescale process model: every channel has its own decay,
  while the update remains an affine scan.

  \item \textbf{Block-diagonal transitions.} Use
  $D_t=\operatorname{blockdiag}(D_t^{(1)},\ldots,D_t^{(m)})$. Small blocks allow local mixing among
  related memory channels while keeping the transition structured enough for parallel
  composition.

  \item \textbf{Basis-rotated diagonal transitions.} Use
  $D_t=R\diag(\alpha_t)R^{\tr}$ with a fixed or slowly changing basis $R$. Forgetting then
  acts in learned memory directions rather than raw coordinates, but the per-token decay
  remains diagonal in that basis.

  \item \textbf{Prefix-conditioned transitions.} Let $D_t$ depend on scan-computable
  statistics $z_t$, such as decayed key energy or an innovation proxy, instead of depending
  directly on the sequential state $S_{t-1}$. Then $D_t=D(x_t,z_t)$ can react to drift while
  keeping $A_t$ available before the scan.
\end{enumerate}

\paragraph{Direction B: parallel covariance and gain families.}
A second route is to approximate the uncertainty $P_t$ with a family whose sufficient
statistics are themselves parallel scans. This keeps the gain adaptive while avoiding the
full Riccati recursion. A key constraint is that the gain should be computed from the
predicted uncertainty, not from a transition-independent rule:
\begin{equation}
  \widehat{P}_t \approx \Pi\!\left(D_tP_{t-1}D_t^{\tr}+Q_t\right),
  \qquad
  \kappa_t =
  \frac{\widehat{P}_t k_t}{r+k_t^{\tr}\widehat{P}_t k_t},
  \label{eq:transition-aware-gain-family}
\end{equation}
where $\Pi$ projects the exact covariance prediction back into a tractable family. Without
this prediction step, the write strength is independent of the forgetting model $D_t$. This
is exactly the coupling ignored by fixed-gain delta-rule models: they let $D_t$ affect the
predicted memory $\widehat{S}_t$, but not the uncertainty that determines the write gain.
\begin{enumerate}[leftmargin=1.8em,itemsep=3pt,topsep=2pt,label=\textbf{B\arabic*.}]
  \item \textbf{Transition-aware scalar covariance.} Set $P_t=c_tI$, but predict the scalar
  uncertainty through $D_t$ before forming the gain. For example,
  $\widehat{c}_t=\rho_t c_{t-1}+q_t$, where $\rho_t$ summarizes the contraction in
  $D_tD_t^{\tr}$, e.g.\ $\rho_t=\alpha_t^2$ for GDN or an average of $\alpha_{t,i}^2$ for
  KDA. The gain becomes
  \begin{equation}
    \kappa_t
    =
    \frac{\widehat{c}_t k_t}{r+\widehat{c}_t\|k_t\|_2^2}
    \equiv \beta_t k_t .
    \label{eq:scalar-transition-aware-gain}
  \end{equation}
  This still recovers the delta-rule direction, but the scalar write strength now knows
  whether the memory has just been strongly forgotten.

  \item \textbf{Diagonal covariance or information form.} Track a diagonal approximation
  $P_t\approx\diag(p_t)$. With diagonal $D_t=\diag(\alpha_t)$, the prediction is
  pointwise:
  \begin{equation}
    \widehat{p}_t = \alpha_t^2\odot p_{t-1}+q_t .
    \label{eq:diag-covariance-prediction}
  \end{equation}
  The resulting per-channel gain is
  \begin{equation}
    \kappa_t
    =
    \frac{\widehat{p}_t\odot k_t}
    {r+\sum_i \widehat{p}_{t,i}k_{t,i}^2},
    \label{eq:diag-kalman-gain}
  \end{equation}
  which is cheap and makes the write direction process-aware. An information-form
  implementation can track $C_t=P_t^{-1}\approx\diag(c_t)$ and recover the same prediction
  by applying the pointwise inverse after the transition step.

  \item \textbf{Block-diagonal covariance.} Track $P_t$ or $C_t=P_t^{-1}$ in small blocks.
  This captures correlations among nearby key channels while replacing one large inverse by
  many small inverses. The prediction remains local:
  $\widehat{P}_t^{(j)}=D_t^{(j)}P_{t-1}^{(j)}D_t^{(j)\tr}+Q_t^{(j)}$.

  \item \textbf{Diagonal-plus-low-rank covariance.} Use
  $P_t=\diag(p_t)+U_tU_t^{\tr}$, with $p_t$ and $U_t$ produced from prefix statistics.
  Woodbury-style reads can keep the gain tractable, while the low-rank term captures global
  uncertainty directions missed by a diagonal approximation.

  \item \textbf{Chunk-boundary covariance updates.} Update $P_t$ only at chunk boundaries,
  while freezing the gain inside each chunk. This gives a TTT-like approximation: the
  expensive uncertainty update is carried by a short sequential pass across chunks, but all
  token updates within a chunk remain fixed-gain affine scans.
\end{enumerate}

The conservative path is to combine these two directions: use a scan-compatible process
model $D_t$ together with a diagonal or block-diagonal information form for uncertainty. The
result is not the exact Kalman filter, but it preserves the two central ingredients of the
optimal update: a predictive memory transition and an uncertainty-weighted residual write.

\section{Background: the Kalman filter}
\label{app:kalman}

The Kalman filter is the optimal recursive estimator for a linear--Gaussian
state-space model; \S\ref{sec:memory-kalman} casts the linear-attention memory as one
instance. We recall the classical form and its specializations here.

\paragraph{Linear--Gaussian state-space model.}
A hidden state $x_t\in\R^n$ evolves and is observed through
\begin{equation}
\begin{aligned}
  x_t &= F_t x_{t-1} + w_t, & w_t &\sim \N(0,Q_t)\quad\text{(process / dynamics)},\\[2pt]
  y_t &= H_t x_t + e_t,      & e_t &\sim \N(0,R_t)\quad\text{(observation)},
\end{aligned}
\label{eq:kf-ssm}
\end{equation}
with transition $F_t$, observation map $H_t$, and independent zero-mean Gaussian
process and measurement noise of covariance $Q_t$ and $R_t$. Given $y_{1:t}$ the
posterior over the state is Gaussian, $p(x_t\mid y_{1:t})=\N(\hat x_t,P_t)$, and the
filter propagates its mean $\hat x_t$ and covariance $P_t$.

\paragraph{The filtering recursion.}
Each step is a \emph{predict} (time update) then an \emph{update} (measurement update):
\begin{equation}
\begin{aligned}
  \text{predict:}\quad & \hat x_t^- = F_t \hat x_{t-1}, & P_t^- &= F_t P_{t-1} F_t^{\tr} + Q_t,\\[2pt]
  \text{update:}\quad  & \hat x_t = \hat x_t^- + K_t\,\nu_t, & P_t &= (I - K_t H_t)\,P_t^-,
\end{aligned}
\label{eq:kf-rec}
\end{equation}
driven by the \emph{innovation} (prediction error) and \emph{Kalman gain}
\begin{equation}
  \nu_t = y_t - H_t \hat x_t^-, \qquad
  K_t = P_t^- H_t^{\tr}\,\Sigma_t^{-1}, \qquad
  \Sigma_t = H_t P_t^- H_t^{\tr} + R_t ,
  \label{eq:kf-gain}
\end{equation}
where $\Sigma_t$ is the innovation covariance. The gain weighs new evidence against
the prior by relative uncertainty: $K_t$ is large when the prior is uncertain ($P_t^-$
large) or the measurement reliable ($R_t$ small), and small otherwise.

\paragraph{Optimality.}
For a linear--Gaussian model \eqref{eq:kf-ssm} the filter is exact: $\hat x_t$ is the
posterior mean, simultaneously the minimum-mean-square-error and maximum-a-posteriori
estimate. Without Gaussianity it remains the best \emph{linear} unbiased estimator. It
is thus the optimal way to fuse a drifting prior with streaming observations.

\paragraph{Special case: recursive least squares.}
Take a \emph{static} state ($F_t=I$, $Q_t=0$) observed through a regressor,
$y_t=\phi_t^{\tr}\theta+e_t$ (so $H_t=\phi_t^{\tr}$, scalar $R_t=r$). Then
\eqref{eq:kf-rec}--\eqref{eq:kf-gain} reduce to \emph{recursive least squares}, which
minimizes $\sum_{\tau\le t}\|y_\tau-\phi_\tau^{\tr}\theta\|_2^2$ online; the covariance
$P_t=\big(\sum_{\tau\le t}\phi_\tau\phi_\tau^{\tr}\big)^{-1}$ is the inverse data Gram
matrix. An exponential \emph{forgetting factor} $\lambda\in(0,1]$ --- weighting old
data by $\lambda^{t-\tau}$ --- is exactly the Kalman filter with process noise
$Q_t=(\lambda^{-1}-1)P_{t-1}$ (equivalently covariance inflation $P_t^-=P_{t-1}/\lambda$),
letting the estimate track slowly drifting parameters. RLS is therefore the
static-state (or random-walk) special case of the Kalman filter.

\subsection{The associative-memory Kalman filter}
\label{app:mem-kf}

\paragraph{State-space model.}
Write the memory itself as the hidden state: it drifts between tokens and is observed, one
noisy key--value pair at a time, through the current key,
\begin{equation}
\begin{aligned}
  S_t &= D_t S_{t-1} + W_t, & W_t &\sim\N(0,Q_t),\\[2pt]
  v_t &= S_t^{\tr}k_t + e_t, & e_t &\sim\N(0,r),
\end{aligned}
\label{eq:mem-ssm}
\end{equation}
with transition/decay $D_t\in\R^{d_k\times d_k}$, drift covariance $Q_t$, and scalar
observation noise $r$. The observation map $k_t^{\tr}$ and the dynamics $D_t$ are shared by
all $d_v$ value channels, so the posterior factorizes across them: we track one estimate
$S_t\in\R^{d_k\times d_v}$ and a single key-space covariance $P_t\in\R^{d_k\times d_k}$.

\paragraph{Optimal solution.}
The minimum-variance estimate is the Kalman recursion \eqref{eq:kf-rec}--\eqref{eq:kf-gain}
specialized to \eqref{eq:mem-ssm}:
\begin{equation}
\begin{aligned}
  \text{predict:}\quad & \widehat{S}_t = D_t S_{t-1}, & \widehat{P}_t &= D_t P_{t-1} D_t^{\tr} + Q_t,\\[2pt]
  \text{update:}\quad  & S_t = \widehat{S}_t + \kappa_t\,(v_t - \widehat{S}_t^{\tr}k_t)^{\tr}, & P_t &= (I - \kappa_t k_t^{\tr})\widehat{P}_t,
\end{aligned}
\label{eq:mem-kf}
\end{equation}
with gain $\kappa_t = \widehat{P}_t k_t/(r + k_t^{\tr}\widehat{P}_t k_t)$. In the static-state limit
($D_t=I$, $Q_t=0$) the estimate has the closed weighted-least-squares form
\begin{equation}
  S_t = P_t \sum_{\tau\le t}\gamma_{t,\tau}\,k_\tau v_\tau^{\tr},
  \qquad
  P_t = \Big(\sum_{\tau\le t}\gamma_{t,\tau}\,k_\tau k_\tau^{\tr}+P_0^{-1}\Big)^{-1},
  \label{eq:mem-wls}
\end{equation}
which \eqref{eq:mem-kf} computes online; the gain $\kappa_t$ is a data- and state-dependent
write strength that replaces the delta rule's fixed $\beta_t k_t$.

\paragraph{Why it resists parallelization.}
The delta-rule family trains in parallel because its per-token update is an \emph{affine}
map of the state, $S_t=A_t S_{t-1}+b_t$, whose coefficients
$A_t=(I-\beta_t k_t k_t^{\tr})D_t$ and $b_t=\beta_t k_t v_t^{\tr}$ depend on the \emph{inputs
alone}; a chunk of such maps composes into dense matmuls (the WY / chunked scan). The
optimal filter \eqref{eq:mem-kf} breaks this on two counts. First, the covariance obeys a
\emph{Riccati recursion} $P_t=(I-\kappa_t k_t^{\tr})(D_t P_{t-1}D_t^{\tr}+Q_t)$ that is
nonlinear (rational) in $P_{t-1}$ and has no closed-form chunk composition --- it must be
iterated token by token. Second, the gain $\kappa_t$ depends on $P_{t-1}$, so the state
coefficient $A_t$ depends on the \emph{accumulated} covariance rather than the current
input, and the recurrence is no longer a fixed linear scan. (Letting $D_t$ or $Q_t$ depend
on $S_{t-1}$ compounds this, making even the predict step nonlinear.) On top of the
sequential dependence, the covariance is $d_k\times d_k$: $O(d_k^2)$ state and work per
step, versus the delta family's $O(d_k d_v)$ matmuls.

\paragraph{Parallelizable relaxations.}
Parallel training needs the per-token coefficients to be either input-only or computable by
an \emph{associative scan} (a prefix operation). Several general relaxations recover this
while keeping some of the second-order/state dependence the delta family discards:
\begin{itemize}[leftmargin=1.4em,itemsep=3pt,topsep=2pt]
  \item \textbf{Structured covariance.} Restrict $P_t$ to a cheap form. A diagonal
  $P_t=\diag(p_t)$ decouples the Riccati into $d_k$ scalar recursions ($O(d_k)$ rather than
  $O(d_k^2)$); a scalar $P_t=c_t I$ collapses it to one recursion and a per-step scalar gain
  $\beta_t=c_t/(r+c_t)$; a diagonal-plus-low-rank $P_t$ interpolates.
  \item \textbf{Gain from prefix statistics.} Replace the exact Riccati by a gain that is a
  pointwise function of \emph{cumulative} quantities a scan can produce --- e.g.\ a running
  key energy $\sum_{\tau\le t}\gamma_{t,\tau}k_\tau k_\tau^{\tr}$ (or its diagonal), or an
  EMA of the innovation. The prefix-sum is parallel; the nonlinearity is applied pointwise.
  \item \textbf{Chunk-wise freezing.} Hold the covariance/gain (and any state-dependent
  transition) fixed within a chunk at its chunk-start value, updating it only across chunks.
  Intra-chunk the update is the fixed-gain delta rule (parallel matmuls); inter-chunk a short
  sequential pass carries $P$. This interpolates between the delta family (never update) and
  the exact filter (chunk size one), mirroring mini-batch test-time training.
  \item \textbf{Learned surrogate.} Replace the Riccati solution by a small learned map from
  cheap, parallelizable features (the input, prefix statistics, the innovation) to the gain
  and transition --- a parametric stand-in for the second-order recursion, trained end to
  end. State-dependent gates are one member of this class.
\end{itemize}
Each trades fidelity to the Kalman optimum for parallelism and cost; the delta family is the
extreme that freezes the covariance entirely, and richer choices move back toward the optimum
along one of these axes.

\subsection{Associative memory as recursive least squares}
\label{app:mem-rls}

\paragraph{Online least-squares objective.}
Dropping the drift ($D_t=I$, $Q_t=0$) reduces the filter of \S\ref{app:mem-kf} to estimating
a \emph{fixed} memory from streaming observations --- ridge-regularized, recency-weighted
least squares,
\begin{equation}
  S_t = \argmin_{S}\ \tfrac12\sum_{\tau\le t}\lambda^{t-\tau}\big\|S^{\tr}k_\tau-v_\tau\big\|_2^2
        + \tfrac12\lambda^{t}\mu\|S\|_F^2 ,
  \label{eq:rls-obj}
\end{equation}
with forgetting factor $\lambda\in(0,1]$ and ridge $\mu>0$ (a decaying prior); this is the
static-state case of \eqref{eq:mem-wls}.

\paragraph{The RLS recursion.}
Zeroing the gradient gives the normal equations $C_t S_t = B_t$ with the \emph{information
matrix} and target
\begin{equation}
  C_t = \sum_{\tau\le t}\lambda^{t-\tau}k_\tau k_\tau^{\tr} + \lambda^{t}\mu I,
  \qquad
  B_t = \sum_{\tau\le t}\lambda^{t-\tau}k_\tau v_\tau^{\tr},
  \label{eq:rls-info}
\end{equation}
both accumulating linearly, $C_t=\lambda C_{t-1}+k_t k_t^{\tr}$ and
$B_t=\lambda B_{t-1}+k_t v_t^{\tr}$. With $P_t=C_t^{-1}$, the Sherman--Morrison identity gives
the classical \emph{recursive least squares} update
\begin{equation}
\begin{aligned}
  g_t &= \frac{P_{t-1}k_t}{\lambda+k_t^{\tr}P_{t-1}k_t}, \\[3pt]
  S_t &= S_{t-1}+g_t\,(v_t-S_{t-1}^{\tr}k_t)^{\tr}, \\[3pt]
  P_t &= \lambda^{-1}\big(P_{t-1}-g_t k_t^{\tr}P_{t-1}\big),
\end{aligned}
\label{eq:rls-rec}
\end{equation}
the exact minimizer of \eqref{eq:rls-obj} computed online.

\paragraph{RLS is online Newton.}
Since $g_t=P_t k_t$, the update \eqref{eq:rls-rec} is a \emph{preconditioned} gradient step on
the per-token recall loss $\mc L_t(S)=\tfrac12\|S^{\tr}k_t-v_t\|_2^2$,
\begin{equation}
  S_t = S_{t-1} - P_t\,\nabla_{\!S}\mc L_t(S_{t-1}), \qquad P_t = C_t^{-1},
  \label{eq:rls-newton}
\end{equation}
whose preconditioner $P_t$ is the inverse accumulated Hessian --- so RLS is \emph{online
Newton} (a natural-gradient step). The delta-rule family replaces $P_t$ by a fixed
$\beta_t I$: \emph{online SGD}, the first-order ($P=I$) approximation of this second-order
($P=C^{-1}$) optimum, with GDN/KDA adding an $\ell_2$ shrinkage (the decay) on top.

\paragraph{Information form: a linear accumulator.}
The obstruction to parallel training is \emph{not} the accumulation: by \eqref{eq:rls-info}
the information matrix $C_t$ (and $B_t$) is a decayed cumulative sum --- a prefix scan, hence
parallelizable. The nonlinearity is confined to the per-step inverse $P_t=C_t^{-1}$ (or
$S_t=C_t^{-1}B_t$). Carrying $C_t$ rather than $P_t$ --- the \emph{information} form --- is
therefore the parallel-friendly representation: the state accumulates linearly, and only the
read-out inverts.

\paragraph{Diagonal RLS is parallelizable.}
Restrict the information matrix to its diagonal, $C_t\approx\diag(c_t)$, with per-channel
accumulated key energy $c_t=\lambda c_{t-1}+k_t\odot k_t$. Both $c_t$ and $B_t$ are decayed
cumulative sums (scans) and the inverse becomes pointwise, so
\begin{equation}
  S_t = \diag(c_t)^{-1}B_t, \qquad\text{i.e.}\quad (S_t)_{i,:}=\frac{(B_t)_{i,:}}{c_{t,i}} .
  \label{eq:diag-rls}
\end{equation}
This is two scans and a per-channel normalization --- fully parallelizable, $O(d_k)$ per step
--- and realizes the gain-from-prefix-statistics option of \S\ref{app:mem-kf}: a per-channel
Newton preconditioner (inverse key energy) that keeps the chunked scan intact, unlike the
full $d_k\times d_k$ RLS whose dense inverse does not.
