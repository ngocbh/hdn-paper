\section{Introduction}
\label{sec:intro}

\begin{itemize}[leftmargin=1.4em,itemsep=5pt,topsep=2pt]

  \item \textbf{Fixed-size memory, edited online.} Linear attention replaces the
  growing key--value cache of softmax attention with a fixed-size recurrent state
  $S_t \in \R^{d_k \times d_v}$, written as tokens stream in and read by the query,
  $o_t = S_t^{\tr} q_t$ \citep{katharopoulos2020transformers}. The central design
  question is how to \emph{edit} that compressed store online --- what to write,
  what to erase, and what to forget.

  \item \textbf{The delta rule is a residual connection along time.} The delta
  update \citep{schlag2021linear,yang2024parallelizing} is
  \begin{equation}
    S_t \;=\; S_{t-1} \;+\; \Fdelta_t(S_{t-1}),
    \qquad
    \Fdelta_t(S) \;=\; \beta_t\, k_t\,(v_t - S^{\tr} k_t)^{\tr}.
    \label{eq:delta-res}
  \end{equation}
  This is exactly a residual block $x_{l+1} = x_l + F(x_l)$ --- but with the block
  index running over \emph{time} rather than \emph{depth}. Memory is a residual
  highway and each token is one block on it: the skip is $S_{t-1}$, and $\Fdelta_t$
  reads the state through $k_t$, forms the prediction error $v_t - S_{t-1}^{\tr}
  k_t$, and writes the correction back along $k_t$.

  \item \textbf{Which optimization problem it solves.} The block $\Fdelta_t$ is
  \emph{one online gradient step}: it descends the local least-squares
  (associative-recall) objective
  \begin{equation}
    \ell_t(S) \;=\; \tfrac12\,\big\| S^{\tr} k_t - v_t \big\|_2^2,
    \qquad
    S_t \;=\; S_{t-1} \;-\; \beta_t\, \nabla_{\!S}\, \ell_t(S_{t-1}).
    \label{eq:opt}
  \end{equation}
  So editing memory along time \emph{is} iterative optimization --- the time-axis
  reading of ``a residual network is unrolled optimization.'' Equivalently, each
  step is the exact minimizer $S_t = \arg\min_S L_t(S)$ of a local objective $L_t$
  that keeps the new state close to the (decayed) previous state while correcting
  the read/write residual \citep{hatamizadeh2026gdn2}.

  \item \textbf{The delta-rule family is a set of choices for that objective.}
  DeltaNet, Mamba-2, Gated DeltaNet, KDA, Mamba-3, and Gated DeltaNet-2 (GDN-2)
  differ only in the local objective $L_t$ and the resulting per-step carry on
  $S_{t-1}$ --- scalar versus channel-wise decay, and a tied versus a decoupled
  erase/write. Table~\ref{tab:fastweight}, ported from GDN-2
  \citep{hatamizadeh2026gdn2}, makes the correspondence explicit.

\end{itemize}
